# 🏠 家校沟通机器人项目 - PRD (Product Requirement Document)

## 1. 项目概述

### 1.1 背景与目标

在校学生受限于手机使用规定，难以及时接收家长的紧急信息。本项目旨在通过 QQ 机器人、中心服务器和 PC 客户端三方联动，实现家长信息到学生桌面的实时、合规投递。

### 1.2 核心技术栈

- **QQ 端：** NapCat (协议层) + Nonebot2 (插件逻辑)
- **消息转发：** Flask + Flask-SocketIO (WebSocket)
- **数据存储：** SQLite3
- **学生端：** PyQt6 + PyQt6-Fluent-Widgets

---

## 2. 角色定义

- **家长：** 通过指定 QQ 群艾特机器人发送消息。
- **学生：** 通过 PC 客户端接收、查阅并确认（已读）消息。
- **管理员（老师）：** 拥有私聊下发消息及控制机器人行为的权限。

---

## 3. 功能需求详述

### 3.1 消息获取与分发 (Plugin & Server)

- **群聊过滤：** 机器人仅处理 `CLASS_GROUP_IDS` 列表中的群消息。
- **普通消息：** 格式为 `@机器人 [文本]`。
- **紧急消息：** 格式为 `@机器人 /紧急消息 [文本]`。
- **私聊消息：** 仅接收 `ADMIN_USERS` 列表中的 QQ 号发来的私聊，视为普通/紧急消息转发。
- **短 ID 生成：** 每次用户使用查询命令查询未读消息时，生成一个5分钟内有效的短id，用于指定撤回和重发的消息。

### 3.2 机器人指令系统 (Plugin)


| 指令     | 格式            | 说明                                |
| ------ | ------------- | --------------------------------- |
| **查询** | `/查询` 或 `/cx` | 机器人回显该发送者发出的所有“未读”消息及对应短 ID。      |
| **撤回** | `/撤回 [短ID]`   | 通知客户端标记该消息为“已撤回”状态（不删除记录），5分钟内有效。 |
| **重发** | `/重发 [短ID]`   | 客户端将该消息置顶，并更新“重发次数”计数器。           |
| **帮助** | `/帮助`         | 自动回复机器人使用指南。                      |


### 3.3 状态反馈逻辑 (Feedback)

当家长发送消息时，机器人根据 Server 记录的状态自动回复：

- **学生在线：** “✅您的普通/紧急消息已转发至学生端。”
- **学生离线：** “❌提示：学生端当前离线，消息已存入服务器，上线后将立即提醒。”
- **考试模式：** “⚠️提示：学生端正处于考试静默模式，消息查看可能延迟。”

### 3.4 客户端核心功能 (Client)

- **时间表控制：** * 普通消息：仅在 `breaks` 规定的时间段内允许主窗口自动弹出。
- 紧急消息：不受时间表限制，立即以 FluentUI 弹窗展示，但不打开主窗口。
- **已读回执：** 学生点击“已读”按钮 -> Server -> Plugin -> 机器人引用原消息并艾特家长：“[回执] 消息已接收”。
- **考试模式：** 开启后客户端进入静默状态，屏蔽所有普通消息弹窗。
- **NTP 校时：** 启动及手动触发时校时，确保弹窗逻辑基于标准北京时间而非系统时间。

---

## 4. UI/UX 界面需求

### 4.1 主窗口 (Main Window)

- **状态栏：** 紧随应用标题，显示 `[🟢 已连接 / 🔴 离线 / 🌙 考试模式]`。
- **侧边导航栏：** * **未读消息页：** 卡片式列表。包含发送者、时间、正文、已读按钮。
- **历史消息页：** 包含已读和未读的所有历史记录，不提供已读按钮，支持状态标记（如：已撤回、重发 x2）。
- **设置页：** 详见下文。

### 4.2 设置页面 (Settings)

- **模式切换：** 考试模式（ToggleSwitch）。
- **网络配置：** 服务器地址、端口、API_KEY。
- **同步管理：** NTP 手动同步按钮、时间表手动刷新按钮。
- **历史管理：** 存储时长选择（1个月/半年/永久等）。
- **个性化：** 开机自启、关闭到托盘、深色/浅色模式切换。

---

## 5. 非功能性需求

### 5.1 安全性

- **内部鉴权：** Plugin 与 Server 通信需校验 `INTERNAL_TOKEN`。
- **数据安全：** 客户端输入及消息展示需过滤 HTML 标签，防止 XSS 注入导致 UI 崩溃。
- **SQL 注入防护：** 使用 SQLAlchemy 参数化查询。

### 5.2 异常处理

- **WS 断线重连：** 客户端需具备自动重连机制，并在状态栏实时反映。
- **消息补发：** 客户端上线后，Server 应自动推送所有状态为“未读”的消息。

---

## 6. 业务流程图 (逻辑摘要)

1. **消息下发：** 家长群聊消息 -> NapCat -> Nonebot(Plugin) -> Server(WS) -> Client。
2. **已读反馈：** Client(点击) -> Server -> Nonebot(Plugin) -> 群聊回复已读。
3. **撤回/重发：** 家长指令 -> Nonebot -> Server -> Client(UI更新)。
