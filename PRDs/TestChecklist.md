# 最终测试清单

## 1. 自动化回归

- `powershell -ExecutionPolicy Bypass -File scripts/smoke_test_all.ps1`
  - 预期：`server / client / plugin` 三项 smoke test 全部通过
- `python scripts/server_roundtrip_smoke.py`
  - 预期：`new_message / query_unread / resend / recall` 全部返回成功
- `python scripts/client_ui_snapshot_probe.py`
  - 预期：客户端可以在无头模式下正常启动、拿到快照、正常退出

## 2. QQ 侧功能

- 群聊中 `@bot 普通消息`
  - 预期：机器人引用原消息回复“消息已存入消息服务器”
- 群聊中 `@bot 紧急消息`
  - 预期：机器人引用原消息回复“紧急消息已存入消息服务器”
- 管理员私聊 bot 发消息
  - 预期：消息可正常入库
- 非管理员私聊 bot 发消息
  - 预期：收到引导去家长群 `@bot` 的提示
- `/查询`
  - 预期：返回当前未读消息及短 ID
- `/撤回 <short_id>`
  - 预期：未读消息可撤回，客户端历史记录正确变为已撤回
- `/重发 <short_id>`
  - 预期：未读消息可重发，客户端卡片置顶，时间显示为首发时间 + 最近重发时间

## 3. Client 未读/历史页

- 新普通消息到达
  - 预期：未读页新卡片渐显，其它卡片顺序下移
- 新紧急消息到达
  - 预期：紧急卡片样式正确，正文更醒目
- 点击“已读”
  - 预期：按钮先进入加载态，成功后卡片淡出退场；失败时保留卡片并提示
- 历史页筛选“全部 / 普通 / 紧急”
  - 预期：全部按时间倒序，不因消息类型打乱顺序
- 已撤回消息
  - 预期：正文显示撤回占位文案，不显示原文

## 4. 回执链路

- 群消息被客户端已读
  - 预期：机器人引用家长原消息并 `@` 家长发送已读回执
- 私聊消息被客户端已读
  - 预期：机器人在私聊中引用原消息发送已读回执
- 紧急消息点击“我知道了（已读）”
  - 预期：直接进入历史记录，不再保留在未读中

## 5. 紧急消息弹窗

- 紧急消息到达时主窗口隐藏在托盘
  - 预期：主窗口和弹窗会被顶出
- 紧急消息到达时主窗口最小化
  - 预期：主窗口恢复，弹窗置顶显示
- 短消息
  - 预期：弹窗正文区高度自适应，不出现大块空白
- 长消息
  - 预期：正文区可滚动查看完整内容
- 点击“延时提醒”
  - 预期：按设定分钟数再次弹出
- 连续两条紧急消息
  - 预期：按队列逐条弹出，不重复、不重影

## 6. 课间逻辑

- break 时间内新普通消息
  - 预期：主窗口自动顶出并切换到“未读消息”页
- break 时间内新紧急消息
  - 预期：主窗口自动顶出并切换到“未读消息”页，同时紧急弹窗正常
- break 时间内未读消息被重发
  - 预期：视为新事件，再次顶窗
- 同一课间连续来多条消息
  - 预期：只在本课间第一次显示“课间休息，有 X 条新消息”

## 7. 时间表与设置

- 切换到合法时间表 JSON
  - 预期：立即生效并写入配置
- 切换到非法 JSON
  - 预期：提示具体错误原因，并自动回退到其它可用 JSON
- 所有 JSON 都非法
  - 预期：提示“没有可用时间表”，课间自动弹出停用
- 重新启动客户端
  - 预期：优先使用上次记录的可用时间表
- 服务器地址 / NTP 地址解锁
  - 预期：需通过验证问题后才能查看或修改；离开设置页后重新上锁

## 8. 稳定性与退出

- 托盘右键“退出”
  - 预期：窗口、托盘图标、Python 进程全部退出
- `Ctrl+C` 结束 `python -m client` / `python -m server`
  - 预期：能正常停止，不残留僵尸进程
- 断网再恢复
  - 预期：客户端持续重连，恢复后自动同步快照
- server 重启
  - 预期：考试模式状态不丢失，客户端可重新连接

## 9. 一周连续运行观察

- 每天检查一次 `client.log` 和 `server.log`
  - 重点看：重复回执、重复弹窗、端口占用、未处理异常、时间表加载警告
- 每天至少测试一次：
  - 普通消息
  - 紧急消息
  - 已读回执
  - 重发
  - 课间顶窗
- 记录以下异常是否出现：
  - 消息重复入库
  - 消息已读后又回到未读
  - 紧急弹窗重复弹出
  - 托盘退出后进程未结束
  - 时间表错误回退失效

## 10. 班级试点建议

- 先只开放给 1 台班级电脑跑
- 先只允许少量家长测试普通消息和紧急消息
- 观察 3 天后再开放 `/重发` 和 `/撤回`
- 试点期间保留 `data/app.db`、`client.log`、`server.log` 备份
